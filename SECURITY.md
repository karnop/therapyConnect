# Security Policy

## Reporting a Vulnerability

The Therapy Connect project takes security seriously and welcomes responsible disclosure of vulnerabilities.

If you discover a security issue, please report it privately using **GitHub Security Advisories**.

### How to Report

1. Go to the repository’s **Security** tab.
2. Click **Report a vulnerability**.
3. Submit the report with detailed reproduction steps and impact.

This allows maintainers to review and fix the issue before public disclosure.

Please **do not open public GitHub issues for security vulnerabilities.**

---

## What to Include in Your Report

To help with triage and remediation, please include:

* A clear description of the vulnerability
* Step-by-step reproduction instructions
* Affected endpoints, files, or components
* Proof-of-concept screenshots or payloads (if available)
* Potential impact and severity assessment
* Suggested remediation (optional)

---

## Responsible Disclosure Guidelines

Security researchers are expected to:

* Avoid public disclosure until a fix has been released
* Avoid accessing, modifying, or deleting other users’ data
* Avoid actions that could disrupt or degrade service availability
* Avoid social engineering, phishing, or physical attacks
* Conduct testing in a non-destructive and ethical manner

---

## Supported Versions

Only the latest version of the repository is actively supported with security updates.

---

## Scope

This policy applies to the source code and deployments of this repository.

Out of scope:

* Third-party services or dependencies not maintained in this repository
* Denial-of-Service or traffic flooding tests
* Social engineering or phishing attacks

---

Thank you for helping keep this project secure.
